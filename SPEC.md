# **消息中心（Notification Center）產品規格文件**

消息中心定位為用戶與系統間的通訊橋樑，規範系統如何透過不同管道與用戶溝通。

## **一、 需求背景與目標**

本系統旨在收攏全服務範圍內多層級的溝通管道，包含：

* **官方對公司客戶**（B2B 訊息、系統公告）  
* **公司對員工**（內部管理通知、政策發布）  
* **員工對員工**（協作提醒、私訊通知）  
* **服務對員工**（自動化流程提醒、任務指派）

透過此中心，我們不僅提供標準化的消息傳遞，更進一步提供**客製化消息分類**與 **Webhook 整合**，作為企業客戶數位轉型之必要基礎建設工具，賦予客戶自主定義通知流程的能力。

## **二、 系統架構**

本系統由四大核心模組組成：

1. **前台設定 (Notification Settings)**：用戶自定義接收偏好與靜音時段。  
2. **前台消息模組 (Notification UI)**：消息列表、紅點提示、彈窗提醒。  
3. **消息後台 (Admin Console)**：規則設定、發送統計、模組管理。  
4. **消息開放服務 (Notification API)**：提供外部業務服務調用的觸發接口。

## **三、 通用消息資料結構 (Data Structure)**

### **1\. 消息主體 (Message Entity)**

| 欄位名稱 | 說明 | 範例/備註 |
| :---- | :---- | :---- |
| **消息 ID (Message ID)** | 唯一識別碼 | UUID |
| **消息類型 (Type)** | 消息所屬業務模組 | 官方訊息、公司公告、招募訊息、表單、排程、便利貼、**任務排程**、客製分類 |
| **消息等級 (Priority)** | 決定是否受靜音限制 | 高 (無視靜音)、中、低 |
| **消息來源類型 (Source Type)** | 觸發者類型 | 服務 (Service) / 會員 (Member) |
| **消息來源對象 (Source ID)** | 具體來源 | Service ID 或 Member ID |
| **消息標題 (Title)** | 顯示標題 | 「您有新的表單待審核」 |
| **消息內容 (Content)** | 訊息內文 | 支援純文字或部分 HTML |
| **消息 Icon (Icon)** | 顯示圖示 | URL 或 Icon 代碼 |
| **跳轉連結 (Link)** | 點擊後的行為 | Deep Link 或 Web URL |

### **2\. 消息管理資料結構 (Management Data)**

| 欄位名稱 | 說明 | 備註 |
| :---- | :---- | :---- |
| **創建時間 (Created At)** | 消息存入 DB 的時間 | \- |
| **發送時間 (Sent At)** | 實際推送到管道的時間 | 若有延遲或排程則異於創建時間 |
| **已讀時間 (Read At)** | 用戶點擊或標為已讀的時間 | Null 代表未讀 |

### **3\. 消息目標對象 (Target & Delivery)**

| 欄位名稱 | 說明 | 範例/備註 |
| :---- | :---- | :---- |
| **目標類型 (Target Type)** | 接收者類型 | 會員 (Member) / 服務 (Service) |
| **目標 ID (Target ID)** | 具體對象 | UID |
| **消息管道 (Channels)** | 多選陣列 | \['System', 'Email', 'Push'\] |
| **固定排程 (Fixed Schedule)** | 循環發送設定 | 每日、每週、無 (即時) |

## **四、 前台設定規格 (Frontend Settings)**

### **1\. 消息推送時段設定 (Push Timing Settings)**

* **四種模式：** 
    * **全開**：所有通知即時發送。
    * **全關**：關閉所有非緊急通知。
    * **接收時段**：僅在指定時段內接收通知（如：09:00 - 18:00）。
    * **批次固定時間**：在固定時間點彙總發送（如：每日 10:00, 16:00）。
* **靜音時段設定 (Mute Period)**：在此時段內，中、低優先級消息將延遲發送。**高優先級通知始終即時發送。**

### **2\. 消息推送渠道設定 (Channel Settings)**

* **L1 消息分類列表**：
    * 官方訊息
    * 公司公告
    * 招募訊息
    * 表單
    * 排程
    * 便利貼
    * 任務排程
    * 客製分類
* **L2 渠道管理 (Channel)**：
    * 系統推播 (System Push)
    * 郵件通知 (Email)
* **設定原則**：依個人偏好獨立設定每個分類的推播渠道。

## **五、 前台消息模組互動 (UI/UX)**

### **1\. 導覽與佈局 (Navigation)**

* **Tab 分類導覽**：依消息類型區分頁籤，支援橫向滑動。  
* **新增按鈕**：右上方「新增分類」Icon，點擊 Popup 輸入名稱，建立客製化 Tab。

### **2\. 通知列表 (Notification List)**

* **展示內容**：消息 Icon / 標題 / 內容 (單行) / 時間戳。

### **3\. 客製分類與 Webhook 功能**

*   **Webhook 整合**：
    *   **專屬 URL**：每個客製分類皆擁有獨立的 Webhook 進入點。
    *   **推送對象範圍 (Target Selection)**：Webhook 配置時可定義預設或動態推送範圍：
        *   **全體推播 (All)**：發送給該租戶下的所有活躍用戶。
        *   **指定人員 (Specific Users)**：透過 `targets` 陣列傳入 `user_id`。
        *   **指定部門 (Specific Departments)**：透過 `targets` 陣列傳入 `dept_id`。
    *   **JSON Payload 規範**：
        ```json
        {
          "title": "消息標題",
          "content": "消息內文...",
          "target_type": "all", // 可選值: all, users, departments
          "targets": [],        // 當類型為 users 或 departments 時必填 ID 陣列
          "link": "https://...", // 選填，點擊後的跳轉連結
          "priority": "medium"  // 選填，預設為 medium
        }
        ```
    *   **安全機制**：提供私有 Token 驗證，支援隨時重置 Token 以確保安全性。

## **六、 系統商後台管理 (System Admin Console)**

系統商後台為管理全局消息分發的核心，分為以下三大管理模組：

### **1. 分類消息設定管理 (Category Configuration)**
*   **分類維護**：支援新增、編輯與刪除消息分類（如：任務排程、表單、客製分類等）。
*   **自動化配置**：
    *   **自動產生 ID**：系統自動分配唯一識別碼。
    *   **自動產生 Webhook**：為每個分類生成專屬的進入點 URL。
*   **安全防護**：
    *   **白名單服務設定**：限制僅允許特定的來源服務（Source Services）調用該分類的 Webhook。
*   **監控與追蹤**：
    *   **用量追蹤**：可依「客群/客戶」、「來源服務」進行維度篩選，查看發送總量。
    *   **錯誤紀錄追蹤**：詳細記錄發送失敗的原因（如：格式錯誤、頻率限制、渠道異常），支援多維度篩選。

### **2. 客戶消息管理 (Customer Management)**
*   **客戶搜尋與列表**：提供強大的搜尋功能，可依「客戶名稱」或「所屬客群」快速定位。
*   **詳情查看**：點擊特定客戶後，可查看該客戶的詳細消息配置與統計：
    *   **消息發送統計 (Usage Statistics)**：
        *   **維度篩選**：可依「來源服務 (Source Services)」篩選該客戶的統計數據。
        *   **累計發送量**：展示該客戶的總發送次數，點擊可查看 **近 30 天發送趨勢圖 (Area Chart)**。
        *   **異常次數**：展示發送失敗的總次數，點擊可查看 **近 90 天異常紀錄分佈圖 (Bar Chart)**。
    *   **分類發送量統計**：在「系統消息分類」與「客製消息分類」列表中，顯示各個分類的獨立累計發送量。
    *   **系統消息分類資訊**：預設分類的開啟狀態與渠道偏好。
    *   **客製消息分類資訊**：該客戶自行建立的客製化 Tab 及其 Webhook 使用狀況。

### **3. 官方消息管理 (Official Announcement)**
*   **消息發送控制**：
    *   **發送渠道**：支援發送至用戶 Email 或前台的「官方公告」分頁。
    *   **目標選定**：可精確選定特定的「客群」或「單一客戶」作為接收對象。
*   **消息主體填寫**：包含標題、內容、Icon、跳轉連結與優先級設定。
